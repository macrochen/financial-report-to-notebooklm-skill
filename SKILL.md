# Financial Report to NotebookLM

## 概述

该 Skill 旨在帮助投资者自动获取全球主要市场（A股、港股、美股）上市公司的财务报告，并无缝集成到 Google NotebookLM 中。它会完成下载、补充最新市场数据、上传、分析师 Persona 配置，并在上传后自动发起一轮结构化分析与研报生成。首次运行会为公司创建专属 notebook，后续同一家公司重跑时会复用原 notebook、增量补传新财报，并基于最新累计资料重新分析。普通公司默认使用《手把手教你读财报》的排雷与三表验证逻辑；若自动识别为商业银行，则会切换到《手把手教你读财报2：18节课看透银行业》的银行股专用分析框架。

## 使用场景

- **A股研究**：用户输入“贵州茅台”或“600519”。
- **美股研究**：用户输入“AAPL”或“NVDA”。
- **港股研究**：用户输入“00700”或“09988”。
- **深度分析**：用户想要基于最新财报进行 AI 辅助的财务分析。

## 估值方法论适用性 (重要)

本 Skill 采用的是“核心盈利能力 (PE) + 净现金调整”的价值投资估值逻辑。为了获得最准确的分析结果，请参考以下适用范围：

### ✅ 最适用场景 (甜点位)
*   **平台型互联网巨头**：如谷歌、腾讯、美团。现金储备大，业务具有网络效应。
*   **高粘性消费品牌**：如开市客 (COST)、茅台、可口可乐。营收预测性强，品牌溢价稳定。
*   **成熟 SaaS/软件公司**：如微软、Adobe。盈利质量高，现金流充沛。
*   **现金牛型企业**：资产轻、回购多、派息稳定的成熟企业。

### ⚠️ 不适用/需谨慎场景 (盲区)
*   **强周期性行业**：如采矿、航运、生猪养殖。利润波动巨大，PE 容易产生误导（巅峰期低 PE 往往是陷阱）。
*   **金融类企业**：如银行、保险。负债结构特殊，不适用“净现金”逻辑，应参考 P/B 估值。
*   **早期/亏损成长股**：如生物医药研发、未盈利科技公司。无 EPS 可供计算，应参考 P/S 或 DCF 估值。
*   **重资产/房地产**：如房企、传统制造业。其价值核心在于资产重置成本 (NAV)，而非年度盈利。

## 深度分析与研报生成机制 (工作原理)

本 Skill 不仅仅是文件的搬运工，它通过以下机制为您提供专家级的深度分析：

### 1. 上下文隔离 (Context Isolation)
首次运行时会为每家公司创建独立的 NotebookLM 笔记本，确保 AI 在分析时拥有“纯净”的知识库，不会受到其他公司或无关信息的干扰。后续重跑同一家公司时，Skill 会复用该 notebook，只增量补传尚未存在的新财报 source，从而在保留历史上下文的同时避免重复上传。
笔记本标题统一采用 `[市场] 代码 名称 - 财报分析`，例如 `[CN] 600519 贵州茅台 - 财报分析`，方便在 NotebookLM 列表中快速查重和定位。

### 2. 角色固化 (Persona Injection)
通过将 `assets/` 中的专家提示词注入为笔记本的 `Persona`，AI 被强制设定为“20 年经验的价值投资分析师”。当前提示词文件使用 Markdown 形式维护（如 `financial_analyst_prompt.md`、`bank_financial_analyst_prompt.md`、`us_financial_analyst_prompt.md`）。普通公司会按《手把手教你读财报》的方法完成排雷与三表验证，再把结论收敛成价值投资备忘录；若标的是商业银行，则会自动切换到银行股专用 Persona，重点分析存款结构、资产质量、拨备、资本充足率、PB 与股利折现。

### 2.5 最新市场数据补充
在上传财报前，Skill 会在线抓取最新市场快照，并生成一份 Markdown 来源文件上传到 NotebookLM，通常包括：
*   **价格维度**：最新股价、昨收、日内区间、52 周区间。
*   **规模维度**：总市值、总股本、流通股本。
*   **估值维度**：PE、PB、目标价等可用字段。
*   **作用**：让 NotebookLM 在做估值、击球区和每股指标判断时，结合最新市场数据，而不是只看历史财报。
*   **增量更新**：同一家公司再次运行时，会先删除旧的市场快照 source，再上传新的快照，避免估值判断引用过期行情。

### 3. 专家级问题生成
Skill 完成上传后，会基于专家模板自动发起“提问-回答”流程：
*   **问题来源**：内置的分析模板会根据公司类型自动切换。普通公司模板涵盖生意模式、护城河、资本配置、财务质量、反方论证、安全边际与击球区；商业银行模板则重点覆盖监管红线、负债端护城河、资产质量、拨备黑匣子、资本与估值，并显式要求结合最新市场数据快照。
*   **用途**：利用 NotebookLM 的 RAG（检索增强生成）能力，强制 AI 进行跨年度、跨报表的对比，并生成带有关联引用的深度研报。
*   **落盘位置**：问答结果、Notebook Summary 和导出的 NotebookLM Report 会保存到 `outputs/<市场>_<输入>/` 目录。
*   **状态持久化**：复用所需的 notebook ID 和 source 列表会保存到 `data/<市场>_<输入>/notebook_state.json`，供下次增量更新使用。

### 4. 证据溯源与持续交互
*   **溯源**：报告中的每一个关键结论（如“续订率 92%”）都可以追溯到财报的原件，点击即可查看出处。
*   **持续性**：笔记本 ID 会返回给用户，并持久化到本地状态文件中。您可以随时进入该笔记本进行针对性的后续追问，Skill 之后重跑时也会默认续用同一个 notebook。
*   **旧标题迁移**：历史上已经创建过的 `xxx 财务深度分析` 风格笔记本，可以通过 `scripts/rename_legacy_notebooks.py` 批量迁移到新标题格式。

## 使用说明

### 1. 身份验证 (首次使用必读)

该 Skill 依赖 `notebooklm-py`。在首次运行前，**必须**在终端手动完成 Google 账号登录：

```bash
cd /Users/shi/workspace/my-skills/.gemini/skills/financial-report-to-notebookllm-skill
.venv/bin/notebooklm login
```
*注：这将打开浏览器窗口。请登录后在终端按回车确认。*

### 2. 运行主脚本

使用虚拟环境中的 Python 运行：

```bash
/Users/shi/workspace/my-skills/.gemini/skills/financial-report-to-notebookllm-skill/.venv/bin/python /Users/shi/workspace/my-skills/.gemini/skills/financial-report-to-notebookllm-skill/scripts/run.py <股票代码或名称>
```

示例：
- **A股**：`.../run.py 600519` 或 `.../run.py 贵州茅台`
- **美股**：`.../run.py TSLA` 或 `.../run.py NVDA`
- **港股**：`.../run.py 00700` (腾讯)

### 3. 反馈结果

向用户提供：
- ✅ 识别到的市场及公司名称。
- 📦 成功下载并处理的报表清单。
- 📚 NotebookLM 链接或 ID。
- 💡 提示已应用针对该市场的专业财务分析模型。
- 🧠 自动生成的分析结果目录（含 Summary、分主题问答、NotebookLM Report）。

## 常见问题

- **识别错误**：如果美股代码被误认，请确保输入的是纯大写字母。
- **未登录 (Auth missing)**：手动运行 `.venv/bin/notebooklm login`。
- **HKEX/SEC 访问受限**：可能需要检查网络代理设置。
